# 官方實價查詢截圖工具

這套工具的目的，是用官方「內政部不動產交易實價查詢服務網」像一般使用者一樣選條件、按搜尋，然後保留官方網頁截圖與擷取表格。截圖可作為主要依據，CSV/JSON 只作為輔助整理。

GitHub 專案位置：<https://github.com/weiwayne12/lvr-comparable-evidence-tool>

## 在新電腦上安裝（從 GitHub 取得）

換一台電腦要使用時，照下面四步做。前兩個工具一台電腦只需各裝一次。

### 步驟 1：先裝好 Node.js 與 Git（各只需一次）

- Node.js：到 <https://nodejs.org> 下載 **LTS 版**安裝。
- Git：到 <https://git-scm.com> 下載安裝。

裝完打開 PowerShell（開始功能表搜尋「PowerShell」），輸入以下兩行確認，各自有出現版本號就成功：

```powershell
node -v
git --version
```

### 步驟 2：把專案抓下來

切換到你想存放的資料夾（例如「文件」），再 clone 專案：

```powershell
cd $HOME\Documents
git clone https://github.com/weiwayne12/lvr-comparable-evidence-tool.git
cd lvr-comparable-evidence-tool
```

### 步驟 3：安裝相依套件與瀏覽器

```powershell
npm install
npx playwright install chromium
```

- `npm install`：裝回 `node_modules`（Playwright 等套件）。
- `npx playwright install chromium`：裝 Playwright 要驅動的瀏覽器。**這步很重要也最常被忘記**，少了它會跑不動。

這兩步都需要網路，第一次會跑比較久，屬正常。

### 步驟 4：執行

雙擊 `點我官方查詢截圖.bat`，或在 PowerShell 輸入：

```powershell
npm run evidence
```

### ⚠️ 重要：個案資料不會跟著 GitHub 一起過去

為保護個資，`.gitignore` 已**刻意排除**下列內容，所以新電腦 clone 下來不會有它們：

- `output/`：截圖與產出資料。
- `案件設定_*.json`：含個資的真實個案設定檔。

新電腦上只會有一份範本 `案件設定.json`。若要在新電腦繼續處理某個個案，請自行用 USB 或加密雲端把該 `案件設定_某案.json` 帶過去，**切勿**為了方便把它 commit 上 GitHub（這是公開 repo，會外洩個資）。

### 之後要更新到最新版

往後在這台電腦只要拉最新版即可，不必重新 clone：

```powershell
cd $HOME\Documents\lvr-comparable-evidence-tool
git pull
npm install   # 若這次更新有新增套件才需要，跑了也無妨
```

## 使用方式

1. 打開 `案件設定.json`，修改縣市、行政區、門牌/社區關鍵字、交易期間等條件。
2. 雙擊 `點我官方查詢截圖.bat`。
3. 第一次執行會安裝 Playwright，需要網路。
4. 完成後，到 `output/evidence/日期_案件名稱/` 查看輸出。

也可以準備多份設定檔（例如 `案件設定_某案.json`），用指令分別執行：

```
node scripts/official_lvr_evidence.js 案件設定_某案.json
```

## 會產生的檔案

- `001_官方查詢條件畫面.png`（已裁切到查詢條件列，清楚顯示你勾選的條件）
- `002_官方查詢結果第1頁.png`
- `官方查詢結果_全部.csv`（逐頁擷取全部結果，與官方畫面一致）
- `官方查詢結果_全部.json`
- `篩選後_排除房地車.csv`（**選用**：`排除房地車` 設 true 時才額外產生的排除版，**不影響分析母體**；分析一律納入房地車）
- `最接近比較標的.csv`（設定 `系爭房屋` 時產生，依接近度排序）
- `比較標的_排名NN.png`（**本工具核心產出**：對最接近的前 N 筆，各自在官方畫面把該列框紅截圖，可直接附卷）
- `操作紀錄.txt`（含截圖 SHA-256 雜湊、門牌彙總、最接近前 5 名、各截圖對應之比較標的）

> 註：結果表是分頁顯示（每頁 15 筆），本工具會自動逐頁點「下一頁」擷取**全部**。`擷取頁數` 設為 `"全部"` 或 `0` 即啟用；設為 `1` 則只抓第 1 頁。

## 設定欄位

- `縣市`：例如 `臺北市`
- `行政區`：例如 `中山區`（留空 `""` 則整個縣市搜尋）
- `門牌社區關鍵字`：填門牌或社區名稱可縮小範圍，例如 `民權東路二段71巷`。留空則不限。
- `交易期間`：民國年月
- `交易標的`：常用為 `["房地"]`（官網「房地」本身已含純房地與房地車）
- `排除房地車`：**預設 `false`（建議維持，納入房地車）**。見下方「為什麼納入房地車」。設 `true` 只會額外多產一份 `篩選後_排除房地車.csv`，分析母體仍含房地車。
- `建物型態`：例如 `["公寓"]`；留空 `[]` 表示不限
- `屋齡`：可填 `不拘`、`0-5`、`5-10`、`10-20`、`20-30`、`30-40`、`40以上`
- `顯示瀏覽器`：`true` 會看得到瀏覽器操作，`false` 則背景執行
- `完成後保留視窗`：`true` 時跑完**不自動關閉瀏覽器**，畫面停在官方查詢結果頁讓你直接檢視、捲動、點選；看完自行關閉視窗，程式才結束（此模式會一律以有畫面方式啟動，不受 `顯示瀏覽器` 影響）。設 `false` 或不設，則照舊跑完自動關閉、只留截圖。

## 重要觀念

### 為什麼納入房地車（不要排除）

官網「房地」這個交易標的，**本身就同時包含「純房地」與「房地(車位)」**。

實測官方資料發現：房地車成交的「**總價／總面積／單價**」**已經是房地部分**，車位的價格與面積是**另外**記在「車位總價」欄。也就是說，含車位成交的**每坪單價本來就不含車位、可直接和純房地比較**，不需要再扣車位總價。

驗證（大直美堤花園，同坪數同格局）：有車位 `47.11坪／3,200萬／單價67.93`，無車位 `47.03坪／3,200萬／單價68.04`——面積一樣、單價一樣。若車位真被灌進總面積，有車位那筆會多出 8~12 坪、單價會被稀釋變低，但並沒有。

因此本工具**預設納入房地車**。排除房地車會白白丟掉大半市場（某些社區七成成交含車位），而且常把**系爭房屋本身**的成交也濾掉。

### 「最接近比較標的」怎麼排序

設定 `系爭房屋` 後，工具會依「接近度」排序比較標的，**分層**由近到遠：
**同棟（同門牌）→ 同巷（本巷，如「161巷」）→ 鄰近他巷／他社區**。
（不同社區單價水準差異大，故同巷優先於鄰巷，避免不同社區混比。）
同層內：同型態（以總樓層分組）→ 交易日越接近**起訴年月**越前。
品質懲罰由重到輕：無單價（全棟交易）→ 特殊/瑕疵交易、一樓 → 有增建（單價偏高）。
含車位者**保留並比較**，「註記」欄會標「含車位(單價已不含車位)」等性質。

排好後，工具會對**最接近的前 `截圖最接近筆數` 筆**，逐一在官方結果頁把該列框紅、各截一張
`比較標的_排名NN.png`，這是可直接附卷的核心產出。排序與標記僅供初步篩選，
最終比較標的之取捨由法院認定。

### 標的價額分析流程（建議）

1. 用系爭房屋**門牌／巷弄**為關鍵字、交易期間涵蓋**起訴時點所在一年內**（要找系爭房屋本身成交可再放寬）。
2. 在 `系爭房屋` 填 `門牌關鍵字`、`樓層`、`起訴年月`（時間錨）。
3. 跑完先看 `最接近比較標的.csv`：**排名 1 是否為同棟**——若系爭房屋本身有成交，即最直接的證據。
4. 採本巷（同巷）成交為主，排除一樓、商業用、超大坪數/複層、增建灌水者，以起訴時點推估**單價區間**。
5. 面積未知不換算總價；訴訟標的價額之核定屬法院職權，僅供參考。

設定範例：

```json
"系爭房屋": {
  "門牌關鍵字": "71巷27號",
  "樓層": 4,
  "建物總樓層": 4,
  "屋齡": 50,
  "起訴年月": "113/5",
  "主建物面積": null
},
"截圖最接近筆數": 5
```

### 用「社區名稱」常常查無資料

實價登錄的「社區簡稱」欄位政府**經常沒有登錄**，所以直接用社區名稱（例如「亞昕首藏」）搜尋，往往查無結果，即使該社區的成交確實存在。**較可靠的作法是用門牌/巷弄搜尋**（例如該社區所在的 `民權東路二段71巷`），就能抓到同棟、同巷的成交。

## 注意

官方網站若改版，欄位代號（DOM id）可能改變，屆時要調整 `scripts/official_lvr_evidence.js`。腳本內已用中文註解標出各欄位對應的 `name`／`id`（如 `ptype`、`f_type`、`avg_var`、`#p_build` 等）。

若卷內需要完整逐筆明細，可再擴充「逐筆點開明細並截圖」與「自動翻頁擷取全部頁次」功能。
